# Requirements Document (REQ)

## 1. Project Overview
**KairiX** is a comprehensive project management and team collaboration tool designed to streamline task tracking, sprint planning, and team operations.

## 2. Functional Requirements
### 2.1 User Management
- User authentication (Signup/Login).
- Role-based access control (Admin/Member).
- Team member capacity management (up to 30 members).

### 2.2 Project & Task Management
- Create and manage projects.
- Task lifecycle management (To Do, In Progress, Review, Done).
- Hierarchical task structure: Epics, Sprints, Tasks, and Subtasks.
- Priority levels and due dates for tasks.

### 2.3 Agile Features
- Backlog management.
- Sprint planning and tracking.
- Epic-level organization of work.

### 2.4 HR & Capacity
- Tracking member availability and capacity.
- Payroll and attendance management (HR specific features).

## 3. Non-Functional Requirements
- **Performance**: Dashboard should load in under 2 seconds.
- **Scalability**: Support for multiple concurrent users and large task sets.
- **Security**: Secure storage of user credentials and sensitive HR data.
- **Usability**: Modern, responsive UI with glassmorphism and dynamic animations.

## 4. User Personas
- **Project Manager**: Plans sprints, assigns tasks, and tracks progress.
- **Developer/Team Member**: Updates task status, views personal workload.
- **HR Administrator**: Manages payroll and team capacity.
